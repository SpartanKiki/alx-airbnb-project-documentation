# Use Case Diagram — Airbnb Clone Backend

This folder contains the Use Case Diagram for the Airbnb Clone backend.

## Diagram overview
Actors:
- Guest — searches and books properties, makes payments, leaves reviews, messages hosts.
- Host — lists properties, manages listings and bookings, responds to guest messages.
- Admin — manages users, properties, and bookings; views system reports.
- Payment Gateway — external system that processes payments and refunds.

Main use cases included:
- Register / Login
- Search Properties
- View Property Details
- Create Booking / Cancel Booking
- Make Payment / Refund Payment
- Create / Edit / Delete Property
- Leave Review
- Send/Respond to Messages
- Admin management functions
